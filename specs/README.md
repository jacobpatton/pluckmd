# harvest specifications

This directory describes the current intended behavior of harvest. The specs are
written at module boundaries rather than individual implementation functions so
they can guide future refactors without freezing internal code structure.

## Design Principles

- harvest must not rely on source-code adapters for specific websites.
- Site support is expressed as runtime `AdapterSpec` data generated by cache,
  heuristics, agent/LLM assistance, or manual review.
- Unknown sites should be handled by observing the page, proposing selectors,
  validating them against live DOM/HTML, collecting links, extracting articles,
  and caching only validated specs.
- The Chrome extension is a local bridge for authenticated browser sessions, not
  a required distribution channel.
- The default workflow must be reproducible in tests without hitting real
  websites.
